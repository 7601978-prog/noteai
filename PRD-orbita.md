## 1. TL;DR

«орбита» — это единый «второй мозг» персонального ассистента, который заменяет стопку бумажных ежедневников, заметок и напоминалок. Один экран, семь блоков (задачи, расписание, дни рождения, мировое время, база контактов, трэвел, сводка дня), тёплый дружелюбный интерфейс и сохранение данных между визитами. Цель — сократить время на «жонглирование» рутиной и высвободить его под действительно важные задачи.

Приложение реализовано как один HTML-файл с локальным сохранением данных (key-value storage), полностью адаптивно под телефон и работает без установки.

---

## 2. Проблема и контекст

Персональные ассистенты, руководители и менеджеры, которые сами ведут свой график, держат рутину в разрозненных местах: бумажный ежедневник, стикеры, заметки в телефоне, напоминания, отдельные списки дней рождения и контактов. Это приводит к типовым болям:

- информация фрагментирована — нет «единого окна» на день;
- задачи не приоритизированы — срочное и важное смешиваются;
- регулярные и долгосрочные дела теряются (нет надёжного напоминания);
- при поездках и работе с разными часовыми поясами легко ошибиться со временем;
- много времени уходит на саму организацию рутины, а не на работу.

**Гипотеза:** объединение ключевых инструментов ассистента в одном лёгком, приятном интерфейсе с сохранением данных снизит когнитивную нагрузку и время на планирование, повысит регулярность использования.

---

## 3. Цели и метрики успеха

### Продуктовые цели
1. Дать ассистенту единое окно на день за один взгляд.
2. Снизить количество «потерянных» задач и забытых дат.
3. Сделать инструмент, в который хочется заходить ежедневно (низкий «порог скуки»).

### Метрики успеха

| Метрика | Целевое значение | Как измеряем |
|---|---|---|
| Возвраты (retention) | Использование ≥ 4 дней в неделю | частота открытий |
| Глубина данных | ≥ 10 задач + ≥ 5 контактов + ≥ 3 ДР через 2 недели | объём сохранённых сущностей |
| Регулярные задачи | ≥ 50% повторяющихся задач закрываются в срок | done-в-срок / всего |
| Субъективная оценка | «экономит время» ≥ 4 из 5 | опрос пользователей |

## 4. Целевая аудитория

**Основная персона — «Ассистент Асель».** Персональный/исполнительный ассистент. Ведёт график одного-двух руководителей, отвечает за встречи, поездки, поздравления, контакты подрядчиков. Живёт в телефоне, ценит скорость и порядок, не любит тяжёлые корпоративные системы.

**Вторичные персоны:**
- Руководитель/менеджер, который сам регулирует свой график и хочет лёгкий личный планировщик.
- Офис-менеджер, координирующий встречи и поездки команды.

**Общие черты:** мобильный сценарий первичен, нужна простота, важны напоминания и работа с часовыми поясами.

## 5. Пользовательские сценарии (Jobs To Be Done)

- Когда я начинаю рабочий день, я хочу за один взгляд увидеть фокус, встречи и ближайшие даты, чтобы сразу понять приоритеты.
- Когда падает новая задача, я хочу быстро положить её в нужный приоритет, чтобы потом не разбирать кучу.
- Когда у меня есть регулярное дело (отчёт, звонок), я хочу, чтобы оно само напоминало о себе по расписанию.
- Когда руководитель в командировке, я хочу видеть его местное время, чтобы не звонить среди ночи.
- Когда приближается день рождения партнёра, я хочу узнать об этом заранее, чтобы успеть подготовить поздравление.
- Когда планируется поездка, я хочу собрать в одном месте даты, чек-лист и справку по направлению.
- Когда мне нужен контакт водителя/поставщика, я хочу найти его за пару секунд.

## 6. Объём и функциональные требования

Приоритеты: **P0** — обязательно для v1.0 (реализовано), **P1** — следующая итерация, **P2** — backlog.

### 6.1. Блок «Сегодня» (сводка дня) — P0
- Приветствие по времени суток + имя пользователя + текущая дата (рус. формат).
- Карточка «Фокус дня»: невыполненные задачи из квадрата «Срочно и важно».
- Карточка «Сегодня в расписании»: события текущего дня по времени.
- Карточка «Скоро дни рождения»: ближайшие даты (≤ 30 дней).
- Карточка «Часы мира»: 2–3 ключевых пояса, живое время.
- Карточка «Ближайшая поездка»: активная/предстоящая поездка с обратным отсчётом.
- Дружелюбные пустые состояния вместо «нет данных».

### 6.2. Блок «Задачи» — матрица приоритетов (Эйзенхауэр) — P0
- Четыре квадрата: «Срочно и важно», «Важно, не срочно», «Срочно, не важно», «Не важно, не срочно».
- Быстрое добавление задачи прямо в нужный квадрат (Enter).
- Свойства задачи: текст, квадрат, статус (выполнено), срок (дата), повтор.
- Повтор: нет / каждый день / каждую неделю / каждый месяц.
- **Логика регулярных задач:** при выполнении повторяющейся задачи она автоматически возвращается со следующей датой (не теряется).
- Долгосрочные задачи с дальним сроком подсвечиваются в «Фокусе дня» по мере приближения.
- Бейджи: срок (с подсветкой просрочки), повтор.
- Перемещение между квадратами через редактирование.
- Счётчик открытых задач по квадрату.

### 6.3. Блок «Расписание» — блоки дня — P0
- Навигация по дням (назад/вперёд, «вернуться к сегодня»).
- Событие: название, время начала, время конца (необяз.), заметка/место, цвет.
- Сортировка по времени, цветовая маркировка.
- Создание/редактирование/удаление события.

### 6.4. Блок «Дни рождения» — P0
- Запись: имя, день, месяц, год (необяз.).
- Расчёт ближайшей даты, обратный отсчёт, возраст (если указан год).
- Сортировка по близости, подсветка ближайших (≤ 7 дней).
- Создание/редактирование/удаление.

### 6.5. Блок «Время» — мировые часы — P0
- Список городов с живым временем (обновление раз в секунду).
- Разница в часах относительно пользователя, иконка день/вечер/ночь.
- Добавление города из преднабора часовых поясов, произвольная подпись.
- Сиды по умолчанию: Астана, Москва, Лондон, Нью-Йорк.
- Удаление города.

### 6.6. Блок «База» — контакты — P0
- Контакт: имя, роль/компания, телефон, email, теги, заметка.
- Поиск по имени, тегу, телефону, заметке.
- Кликабельные телефон (`tel:`) и почта (`mailto:`).
- Аватар с инициалом и цветом, чипы тегов.
- Создание/редактирование/удаление.

### 6.7. Блок «Трэвел» — поездки — P0
- Поездка: город, страна, даты начала/конца, часовой пояс направления, цель, заметка.
- Обратный отсчёт и фазы: предстоит / в поездке / завершена.
- Местное время в точке назначения (живое).
- Чек-лист подготовки (виза, билеты, чемодан…) с отметками.
- **AI-справка по направлению** (через Claude API): валюта и курс к тенге, виза для граждан РК, тип розеток, погода по сезону, деловой этикет, полезные фразы. Кэшируется, можно обновить.
- Создание/редактирование/удаление.

| **Локализация** | Интерфейс и форматы дат — на русском (`ru-RU`) |
| **Отказоустойчивость** | Все операции хранения и AI-запросы обёрнуты в обработку ошибок; дружелюбные сообщения при сбое |

## 8. Информационная архитектура

```
орбита
├── Шапка: лого + имя пользователя (настройка приветствия)
├── Навигация: горизонтальные вкладки (скролл на телефоне)
└── Экраны:
    ├── Сегодня        (агрегация всех блоков)
    ├── Задачи         (матрица 2×2)
    ├── Расписание     (день, листание)
    ├── Дни рождения   (список + отсчёт)
    ├── Время          (мировые часы)
    ├── База           (контакты + поиск)
    └── Трэвел         (поездки + чек-лист + AI-справка)
```

Редактирование сущностей — через нижнюю модальную панель (bottom sheet) на телефоне, центрированную на десктопе.

## 9. Модель данных

Хранение по ключам (key → JSON):

| Ключ | Структура |
|---|---|
| `settings` | `{ name }` |
| `tasks` | `[{ id, text, quad, done, due, rep }]` — `quad ∈ {ui, iu, un, nn}`, `rep ∈ {none, day, week, month}` |
| `events` | `[{ id, title, start, end, note, color, date }]` |
| `bdays` | `[{ id, name, day, month, year? }]` |
| `clocks` | `[{ id, label, tz }]` — `tz` = IANA-зона |
| `contacts` | `[{ id, name, role, phone, email, tags[], note }]` |
| `trips` | `[{ id, dest, country, start, end, tz, purpose, note, checklist:[{text,done}], briefing }]` |

## 10. Дизайн и UX-принципы

- **Тон:** тёплый, «бумажный», дружелюбный — чтобы инструментом хотелось пользоваться ежедневно, без ощущения корпоративной системы.
- **Палитра:** кремовый фон, коралловый/шалфейный/янтарный/лиловый акценты по квадратам приоритетов.
- **Типографика:** характерный дисплейный шрифт для заголовков + чистый читаемый для текста (полная поддержка кириллицы).
- **Микровзаимодействия:** мягкие тени, скругления, лёгкие анимации появления, «пульс» у ближайших дней рождения.
- **Пустые состояния:** не «нет данных», а доброжелательные подсказки и эмодзи.
- **Принцип «один взгляд»:** экран «Сегодня» должен отвечать на вопрос «что важно прямо сейчас» без скролла.

## 11. AI-функции

**Справка по направлению (Claude).** По кнопке в карточке поездки приложение запрашивает у модели компактную деловую справку по городу назначения (валюта, виза для РК, розетки, погода, этикет, фразы). Результат кэшируется в данных поездки и обновляется по запросу. При отсутствии связи — корректное сообщение об ошибке без падения интерфейса.

_Потенциал развития:_ авто-разбор задачи в нужный квадрат, генерация текста поздравления, краткая сводка дня для пересылки руководителю.

## 12. Граничные случаи

- Имя пользователя не задано → нейтральное приветствие, кнопка «Кто вы?».
- Пустые блоки → дружелюбные заглушки.
- Просроченные задачи → подсветка бейджа срока.
- ДР сегодня → отметка «сегодня 🎂», отдельная индикация.
- Поездка в прошлом → фаза «завершена», уводится вниз списка.
- Сбой хранилища / AI-запроса → перехват ошибки, сохранение работоспособности.
- Один и тот же часовой пояс под разными подписями (Астана/Алматы) → допустимо.
